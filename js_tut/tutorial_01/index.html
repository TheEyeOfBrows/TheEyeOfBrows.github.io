Setting up and Hello World!

First we need to set up an environment for writting and running JavaScript.
I'm choosing GitHub's Atom, which is a pretty nice IDE for JavaScript.
Download Atom from atom.io.

We're going to add one non-default package to Atom - browser-plus, which adds an internal web browser to atom, specifically with the ability to run live previews of our project.
Once installed, open the File, Settings menu. In the newly opened tab, select Install, search for browser-plus, and install it.
Close the settings tab when done. More on browser-plus later.

Now we're going to set up a project.
Close all your open tabs in Atom, and open File, Open Folder. Navigate to where you'd like your project to live and create a new folder there (right-click menu) with the project name (tutorial 1), then select that folder and click on the Select Folder button.

Right-click the project folder and select New File. Name the file index.html. index.html is a good main page name, because it's used as a default name in all web servers.
Open up that new html file and drop in the following text. 

<!DOCTYPE HTML>
<html>
<head>
    <script>
    alert('Hello World!');
    </script>
</head>
<body>
    A webpage
</body>
</html>

Now, let's open up the HTML page in the browser-plus browser.
The key combo to open the current file in a new browser tab is CTRL + ALT + o

So what just happened? In the HTML page we just ran, we had a script block run the alert function, which opens a message box with the text "Hello World!". This funciton is run by JavaScript for the client using the page!

Let's break down what's in this HTML file.

First look at the first line, <!DOCTYPE HTML>. This HTML element is extra information (meta-data, identified by the exclamation point) for the browser to know how to display the page. That flag lets the browser know we're using HTML5, which gives us access to some newer HTML features like the <script> shorthand element (older HTML files need to specify what language we're using) along with other features we're not yet taking advantage of.

Now we have the HTML document definition itself. We start with the root <html> element, which wraps the entire page. The <html> element contains two elements, the <head>, and the <body>.

The <head> is where we define things that aren't content, like the page title, the links to style documents (css), and external script files (javascript).
The <body> is where we set up the visible content of the page, like text and images.
In our page, a <script> element written in the <head> runs that javascript alert function once as the page loads.
In the body of the page, our <body> simply has the pain text A webpage to show the user.

Now let's look at the javascript.

Inside the <script> element, we have the line of code alert('Hello World!'); .

  • alert is the name of the function we want to run
  • The parentheses ALWAYS follows the name of the function we want to run
  • The parentheses also contains the parameters we want to give to the alert function
  • We want to use the text Hello World! as the parameter
  • Text needs to be wrapped in either single quotes ' or double quotes "
  • Single quotes means we DON'T want special characters in our text, and that it should be used as written

To elaborate on that last point, sometimes in text you may want to use two lines of text. The new-line character is \n.

Try changing the <script> element contents to the following.
alert("Line 1\nLine 2");
It's important to notice that now we're using the double quotes.
Refresh the page and see how now you get text seperated by a new line.

Let's try a slightly different HTML page.
Replace the entire file with the following code.

<!DOCTYPE HTML>
<html>
<head>
</head>
<body>
    <script>
    var newElement = document.createElement('p');
    newElement.innerText = 'Hello World!';
    document.body.appendChild(newElement);
    </script>
</body>
</html>

Save and refresh the page.
You'll see that instead of popping up the alert message box to display the text, we're writting the text directly to the page! (alert messages, while useful when developing, are the worst thing for users to ever see)

The new script we wrote does the following:

First, we create a new HTML element (also known as a DOM element). We use the document object (which represents the entire html page) to create a p element (for paragraph). This new element is stored into the var newElement variable, so we can use it in the following lines of code.

Second, we assign some text to the newElement element's innerText field, so that it has something to show the user. Once again, the text we want to use must be surrounded in quotes.

Third and finally, we use appendChild to add the new element to the bottom of the document's body element (remember that the html document can only contain a head and a body element). The appendChild function puts the new element in the last position of the body element. Note that for this function, we're passing in a variable instead of text, so we don't need to use quotes here.

One question left is why did we move the script out of the <head> element and into the <body> element?

That's because HTML documents are run IN ORDER of what's written, with the head being executed before the body. That means that if the script is executed in the <head> element, the <body> element doesn't even exist yet because that's further down the page!
Know that this is true only for the HTML part of the page, not the JavaScript itself. Later on, you'll learn about ways of dealing with this issue.

This is a good time to try experimenting.

In the browser tab, try enabling the lightning-bolt button, which enables Live Preview. Live Preview will refresh the browser tab any time the html page is saved, so you can see your changes instantly.

Try changing the hello world text to instead be a number (without quotes), and try doing math with it before assigning it to the newElement. See the links at the bottom for more info about variables.
